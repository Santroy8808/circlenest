# Naming And Legacy Notes

- Legacy `Bazaar` code and `BazaarListing` models should become `Market` concepts in user-facing and new domain code.
- Legacy `PRO` / `Biz` language should normalize to `Professional` unless the product owner chooses a different final tier label.
- Chat and Mail must be distinct modules.
- Admin is a role, not a subscription tier.
- Auditor is an account capability/tier with specific profile creation rights.
